# StepCore — архитектурный канон

Актуально: V10.0 (versionCode 63), 08.07.2026.
Полная версия канона — в Claude Project (DoroKart.md).

## Ключевые архитектурные решения

1. **Чип — единственный источник счёта** (V9.0). TYPE_STEP_COUNTER считает
   всегда; StepDetector только классифицирует WALK/RUN. Основание: тапы
   надували детектор в 3-5 раз, чип дал 0 ложных. Регресс: 96-98/100 во
   всех режимах, 0 тапов, реальная поездка на машине без ложных шагов.

2. **Взаимная коррекция источников** (V9.2). Тряска: чип врёт, детектор
   честен -> shake-вето отбрасывает дельты чипа. Залипшая метка TRANSPORT:
   >=5 шагов чипа под меткой снимают её.

3. **Снапшоты дней** (V9.9, V9.19). Калории и дистанция закрытого дня
   заморожены в DayRecord. Смена веса или калибровки не переписывает
   прошлое. Все экраны читают снапшоты.

4. **Энергия раздельно**: Active (LCDA + Margaria, с грузом) и Basal
   (Mifflin-St Jeor, 24/7 от календарного времени). Total = Active+Basal.
   Покой начисляется только за прожитое время с данными.

5. **Длина шага**: SL = a*каденс + b. Уровни ESTIMATE -> MANUAL -> GPS.
   GPS-калибровка через чистый LocationManager (не Maps, не Fused),
   координаты не сохраняются.

6. **Реестр калибровок** (V10). Веса: длина шага 50%, темп ходьбы 35%,
   темп бега 15%. Свежесть падает 100% -> 70% за 30 дней.

## Правила, выведенные из провалов

- Любое блокирующее состояние обязано иметь путь выхода, не зависящий от
  отсутствия того сигнала, который оно блокирует (deadlock блока чистоты
  V7.11 и транспорта V8.x).
- Ядро меняется только по собранным данным, не по гипотезам.
- Не показывать метрику, которая врёт: лучше отсутствие цифры.
- В Kotlin raw-строках escape-последовательности не работают.
- Файл с большим числом правок отдавать полной перезаписью.

## Барометра на устройстве НЕТ

Отклонено окончательно: лестницы, этажи, счёт ступеней, уклон/поверхность.
GPS-высота (±20 м) для этажа (3 м) непригодна.

## Roadmap

V11: гистерезис метки WALK/RUN (чинит флаппинг) + Confidence Score с
причинами. Confidence — качество МЕТКИ, не счёта.

V12: детект дрейфа калибровки (медиана интервала vs эталон, 7 из 10 дней);
персональная модель походки, обучение только из high-confidence сессий.
Нельзя обучать предохранители (гиро-капы, транспортные пороги).

V13+: фантомный преследователь, фото в журнале (photoUri готов), трек
прогулки, фильтры-запросы, живая диагностика, виджеты.

## Известные хвосты

- Флаппинг меток Бег/Ходьба (счёт не портит, лечится в V11).
- Тряска при заблокированном экране считается (детектор молчит).
- Техдолг: sortedArray() медианы на каждом сэмпле акселя; Stats читает
  prefs при каждом вызове.
- Дни до V9.19 заморожены с текущей калибровкой, не с исторической.

## Экспедиция v112: радар окрестностей

Экран знания, а не экран мира. Полярный: лагерь в центре, кольца дальности,
конус запаха по подветренному румбу, метки зверей по ПОСЛЕДНЕМУ наблюдению.

Правила, которые теперь канон:

- **Наблюдение != позиция.** Радар показывает то, что человек мог узнать:
  увидел (румб точен), прочитал след (дальность — догадка), услышал (и то
  и другое на слух). Наблюдение рождается там же, где строка журнала:
  радар не знает того, о чём журнал промолчал (инвариант
  `radar.no_orphan_observations`).
- **Туман войны выведен, а не нарисован.** Неопределённость = ошибка
  источника + снос зверя со временем (волк 2,4 км/сутки, лось 0,7).
  Больше 9 км — метка гаснет: размытое враньё всё равно враньё.
- **Красный = зверь чует лагерь сегодня** (в конусе запаха или ближе 2 км).
  Единственное на экране, что требует решения.
- **Ничего не хранится.** Наблюдения пересчитываются из seed, как и погода.
  Радар работает у старых и архивных экспедиций.
- **Отпечаток мира заморожен** (`world.fingerprint_frozen`): хеш событий
  50 сидов x 100 дней. Новый слой, сдвинувший поток случайности старых
  миров, теперь падает в песочнице, а не в реальном журнале.
